# Release Notes - Versão 1.2.0

Foco em detalhes básicos de perfil do usuário autenticado: novos endpoints para obter e atualizar perfil, validações de dados e documentação atualizada.

## Visão Geral

Esta versão introduz endpoints de perfil para o usuário autenticado, incluindo obtenção do perfil e atualização via multipart (com avatar opcional). Também inclui o endpoint `/auth/me` para dados essenciais de identificação e papel (role), com lista de turmas ativas para estudantes.

---

## Funcionalidades Principais

### Perfil do Usuário

- `GET /user/profile`: retorna `id`, `name`, `email`, `avatarUrl`, `birthDate` e `description` do usuário autenticado
- `PATCH /user/profile` (multipart): atualiza nome, data de nascimento e descrição; permite remover descrição e avatar; aceita upload de novo avatar

### Dados do Usuário Autenticado

- `GET /auth/me`: retorna `userId` e `role`; para `STUDENT`, inclui lista de turmas ativas (`id`, `className`)

---

## Detalhes Técnicos

- DTOs: `ProfileResponseDTO`, `EditProfileRequestDTO`, `UserProfileDTO`, `ClassroomInfoDTO`
- Services: `UserService.updateAuthenticatedProfileUser`, `UserService.getAuthenticatedUserProfile`, `AuthService.getUserProfile`
- Segurança/Validação:
  - Avatar: tipos permitidos `image/png`, `image/jpeg`, `image/jpg`; tamanho máximo `3MB`
  - Validação do conteúdo com `ImageIO.read(...)` para rejeitar arquivos inválidos
  - Mensagem de erro atualizada: "Formato inválido. Use: PNG, JPG ou JPEG"
- Auth: uso de `CustomUserDetails` para obter `id` e `role` do usuário autenticado

---

## Notas

- Sem breaking changes
- Sem alterações no banco de dados
- Testes ajustados para padronizar chamadas `verify(...)` sem `times(1)`
- Documentação OpenAPI atualizada nos endpoints de perfil
