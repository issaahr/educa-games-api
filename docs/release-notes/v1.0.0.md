# Release Notes - Versão 1.0.0

Primeira versão estável da API EducaGames, uma plataforma REST para ensino de programação desenvolvida como projeto final para o Programa +PraTI.

## Visão Geral

Esta versão consolida todas as funcionalidades desenvolvidas até o momento, estabelecendo uma base sólida para o sistema de ensino de programação. A API oferece autenticação segura, gerenciamento de usuários via convites e infraestrutura completa para desenvolvimento e produção.

---

## Funcionalidades Principais

### Autenticação e Segurança

**Sistema de autenticação JWT**

- Login com validação de credenciais
- Logout com invalidação de cookies
- Tokens JWT armazenados em cookies HttpOnly para segurança
- Suporte a tokens expirados e inválidos
- Endpoint `/auth/me` para obter dados do usuário autenticado

**Autorização baseada em roles**

- Sistema de papéis com três níveis: ADMIN, INSTRUCTOR, STUDENT
- Controle de acesso granular por endpoint usando `@PreAuthorize`
- ADMIN: gerencia instrutores e convites de instrutores
- INSTRUCTOR: gerencia turmas, alunos e convites de estudantes
- STUDENT: acesso a recursos de turmas (futuro)
- Integração com Spring Security

**Filtros de segurança**

- `JwtFilter` para validação automática de tokens em requisições
- `StrictOriginFilter` para validação de origem em produção
- Configuração centralizada de endpoints públicos

**Configurações de segurança**

- CORS configurável por ambiente
- Validação de origem para endpoints sensíveis
- Cookies seguros em produção (Secure, HttpOnly, SameSite)
- CSRF desabilitado para API stateless

### Sistema de Convites

**Gerenciamento de convites**

- Criação de convites com tokens únicos (UUID)
- Envio automático de emails com templates HTML responsivos
- Validação de convites antes do cadastro
- Sistema de status completo: NOT_SENT, AWAITING_ACCEPTANCE, ACCEPTED, EXPIRED, REVOKED
- Expiração configurável de convites (padrão: 24 horas)
- Reenvio de convites com controle de tempo (mínimo 2 horas entre reenvios)
- Contador de reenvios e timestamp do último reenvio
- Associação de convites a turmas (para convites de estudantes)

**Hierarquia de convites**

- ADMIN pode convidar INSTRUCTOR (sem associação a turma)
- INSTRUCTOR pode convidar STUDENT para suas turmas (classroomId obrigatório)
- Validação de permissões por role
- Controle de acesso: cada role só gerencia seus próprios convites

**Cadastro por convite**

- Endpoint `/auth/validate-invite` para validar tokens
- Endpoint `/auth/complete-signup` para finalizar cadastro
- Validação de expiração e uso único de convites
- Associação automática de role baseada no convite
- Inclusão de nome da turma no convite (para estudantes)

**Operações de convites**

- Listagem com paginação, busca e ordenação
- Exclusão de convites (com validação de permissões)
- Reenvio com proteção contra spam (2 horas)

### Sistema de Emails

**Envio de emails transacionais**

- Suporte a SMTP local (MailHog) para desenvolvimento
- Integração com API Resend para produção
- Templates HTML responsivos e personalizáveis
- Sistema de templates baseado em interface (`EmailTemplate`)
- Template específico para convites (`InviteEmailTemplate`)

**Configuração de emails**

- Suporte a múltiplos provedores
- Configuração por perfil (dev/prod)
- Logs detalhados de envio e falhas

### Gerenciamento de Usuários

**Modelo de usuário**

- Entidade User com campos: nome, email, senha, role, status ativo
- Suporte a data de nascimento
- Criptografia de senhas com BCrypt
- Soft delete através de BaseEntity
- Três roles: ADMIN, INSTRUCTOR, STUDENT

**Operações de gerenciamento (ADMIN)**

- Listagem de instrutores com paginação, busca e ordenação
- Filtro por status (ativo/inativo)
- Busca por nome ou email
- Exclusão permanente de instrutores
- Alteração de status (ativo/inativo) de usuários/instrutores

**Autenticação de usuários**

- Validação de credenciais
- Geração de tokens JWT personalizados
- Carregamento de detalhes do usuário para contexto de segurança
- Obtenção de perfil do usuário autenticado

### Sistema de Turmas

**Gerenciamento de turmas (INSTRUCTOR)**

- Criação de turmas associadas ao instrutor autenticado
- Listagem de todas as turmas do instrutor
- Visualização de detalhes completos de uma turma
- Inclusão de lista de alunos na visualização de detalhes
- Status ativo/inativo para turmas

**Modelo de dados**

- Entidade Classroom: nome, status ativo, instrutor
- Entidade StudentClassroom: matrícula, status ativo, relacionamento aluno-turma
- Relacionamento muitos-para-muitos entre User (STUDENT) e Classroom
- Constraint única: um aluno não pode estar duplicado na mesma turma

**Funcionalidades**

- Turmas vinculadas automaticamente ao instrutor criador
- Validação de propriedade (instrutor só acessa suas próprias turmas)
- Lista de alunos com informações: nome, matrícula, status

### Arquitetura e Infraestrutura

**Estrutura em camadas**

- Controllers REST: AuthController, UserController, InviteController, ClassroomController
- Services com lógica de negócio: AuthService, UserService, InviteService, ClassroomService, EmailService
- Repositories JPA: UserRepository, InviteRepository, ClassroomRepository
- DTOs para transferência de dados

**Configuração por perfis**

- Perfil `dev`: logs detalhados, ddl-auto=update, SMTP local

**Containerização**

- Dockerfile multi-stage otimizado
- Docker Compose para ambiente completo (API + PostgreSQL)
- Build otimizado com cache de dependências

**Utilitários JWT**

**Utilitários de Cookies**

- Criação de cookies HttpOnly seguros

**Utilitários de Resposta**

### Banco de Dados

**Persistência**

- PostgreSQL 15 como banco de dados
- Spring Data JPA para acesso a dados
- Entidades com auditoria (createdAt, updatedAt)
- Relacionamentos e constraints configurados

**Modelo de dados**

- Tabela `users` para usuários (ADMIN, INSTRUCTOR, STUDENT)
- Tabela `invites` para convites (com relacionamento com turmas)
- Tabela `classrooms` para turmas
- Tabela `student_classes` para relacionamento aluno-turma (com matrícula)
- BaseEntity para campos comuns de auditoria (createdAt, updatedAt)
- Constraints únicas: email único em users, token único em invites, email+classroom único em invites

### Monitoramento

**Health Checks**

- Endpoint `/actuator/health` para verificação de saúde
- Integração com Spring Boot Actuator
- Verificação de dependências (banco de dados)

### Tratamento de Erros

**Sistema de exceções**

- Exceções customizadas por tipo de erro com códigos HTTP apropriados:
  - `BadRequestException` (400) - Dados inválidos
  - `UnauthorizedException` (401) - Não autenticado
  - `ForbiddenException` (403) - Sem permissão
  - `NotFoundException` (404) - Recurso não encontrado
  - `ConflictException` (409) - Conflito (ex: convite já usado)
  - `InviteExpiredException` (410) - Convite expirado
  - `JwtExpiredException` (401) - Token JWT expirado
  - `JwtInvalidException` (401) - Token JWT inválido
- `GlobalExceptionHandler` para tratamento centralizado
- Respostas padronizadas de erro (ErrorResponse)
- Tratamento de erros de validação (MethodArgumentNotValidException) com lista de erros

**Validação de dados**

- Validação automática de DTOs com Bean Validation
- Mensagens de erro descritivas
- Validação de campos obrigatórios e formatos

### Configuração

**Gerenciamento de variáveis de ambiente**

- Suporte a arquivo `.env` com Dotenv Java
- Configuração separada por perfil
- Valores sensíveis externalizados

**Configurações de segurança**

- `SecurityConfig` para políticas de autenticação
- `CorsConfig` para configuração de CORS
- `PublicEndpoints` para centralização de rotas públicas
- `CustomAuthenticationEntryPoint` para respostas de erro personalizadas

### Endpoints Disponíveis

**Autenticação** (`/auth`)

- `POST /auth/login` - Autenticação do usuário
  - Requer: `LoginRequestDTO` (email, password)
  - Retorna: `204 No Content` (token em cookie HttpOnly)
  - Erros: `400` (validação), `401` (credenciais inválidas)
  - Público: Sim

- `POST /auth/logout` - Logout do usuário
  - Retorna: `204 No Content`
  - Público: Sim

- `GET /auth/me` - Dados do usuário autenticado
  - Retorna: `200 OK` com `UserProfileDTO` (userId, name, role)
  - Erros: `401` (não autenticado)
  - Público: Não (requer autenticação)

- `GET /auth/validate-invite?token={token}` - Validação de token de convite
  - Parâmetros: `token` (query param, UUID)
  - Retorna: `200 OK` com `InviteDetailsResponseDTO` (email, role, className)
  - Erros: `404` (inválido), `409` (já usado), `410` (expirado)
  - Público: Sim

- `POST /auth/complete-signup` - Finalização de cadastro via convite
  - Requer: `CompleteSignupRequest` (token, name, password)
  - Retorna: `201 Created`
  - Erros: `400` (validação/convite não encontrado), `409` (já utilizado), `410` (expirado)
  - Público: Sim

**Usuários** (`/user`) - Requer role: ADMIN

- `GET /user/instructors` - Lista instrutores com paginação e busca
  - Parâmetros: `active` (boolean, obrigatório), `page`, `size`, `search`, `sortBy`, `sortDir`
  - Retorna: `200 OK` com `PageResponseDTO<ListInstructorDTO>`
  - Erros: `401` (não autenticado), `403` (sem permissão)
  - Público: Não (requer role ADMIN)

- `DELETE /user/instructors` - Exclui um instrutor permanentemente
  - Requer: `OnlyIdDTO` (id)
  - Retorna: `204 No Content`
  - Erros: `400` (não é instrutor), `401`, `403`, `404` (não encontrado)
  - Público: Não (requer role ADMIN)

- `PUT /user/status` - Altera status (ativo/inativo) de usuário/instrutor
  - Requer: `ChangeUserStatusDTO` (id, status)
  - Retorna: `204 No Content`
  - Erros: `401`, `403`, `404` (usuário não encontrado)
  - Público: Não (requer role ADMIN)

**Convites** (`/invite`) - Requer role: ADMIN ou INSTRUCTOR

- `POST /invite/send?classroomId={id}` - Envia um convite por email
  - Parâmetros: `classroomId` (opcional, obrigatório para INSTRUCTOR)
  - Requer: `CreateInviteRequestDTO` (email)
  - Retorna: `200 OK`
  - Erros: `400` (validação/turma não encontrada), `401`, `403`, `404` (turma não encontrada), `409` (convite já existe)
  - Público: Não (requer role ADMIN ou INSTRUCTOR)
  - Nota: ADMIN convida INSTRUCTOR (classroomId ignorado). INSTRUCTOR convida STUDENT (classroomId obrigatório)

- `GET /invite?classroomId={id}` - Lista convites com paginação e busca
  - Parâmetros: `classroomId` (opcional, obrigatório para INSTRUCTOR), `page`, `size`, `search`, `sortBy`, `sortDir`
  - Retorna: `200 OK` com `PageResponseDTO<InviteDTO>`
  - Erros: `400` (classroomId obrigatório para INSTRUCTOR), `401`, `403`
  - Público: Não (requer role ADMIN ou INSTRUCTOR)

- `DELETE /invite` - Exclui um convite
  - Requer: `OnlyIdDTO` (id)
  - Retorna: `204 No Content`
  - Erros: `401`, `403`, `404` (convite não encontrado)
  - Público: Não (requer role ADMIN ou INSTRUCTOR)

- `POST /invite/resend` - Reenvia um convite existente
  - Requer: `OnlyIdDTO` (id)
  - Retorna: `204 No Content`
  - Erros: `400` (reenviado recentemente - menos de 2 horas), `401`, `403`, `404`
  - Público: Não (requer role ADMIN ou INSTRUCTOR)

**Turmas** (`/classroom`) - Requer role: INSTRUCTOR

- `POST /classroom/create` - Cria uma nova turma
  - Requer: `CreateClassRequestDTO` (name)
  - Retorna: `201 Created` com dados da turma criada
  - Erros: `400` (validação), `401`, `403`
  - Público: Não (requer role INSTRUCTOR)

- `GET /classroom/` - Lista todas as turmas do instrutor autenticado
  - Retorna: `200 OK` com `List<ClassroomDTO>` (id, name, active)
  - Erros: `401`, `403`
  - Público: Não (requer role INSTRUCTOR)

- `GET /classroom/{id}` - Obtém detalhes completos de uma turma
  - Parâmetros: `id` (path variable)
  - Retorna: `200 OK` com `ClassroomDetailsDTO` (id, name, students)
  - Erros: `401`, `403`, `404` (turma não encontrada ou não pertence ao instrutor)
  - Público: Não (requer role INSTRUCTOR)

**Monitoramento**

- `GET /actuator/health` - Health check da aplicação
  - Retorna: Status da aplicação e dependências
  - Público: Sim

**Documentação**

- `GET /swagger-ui.html` - Interface Swagger UI
  - Documentação interativa da API
  - Público: Sim

- `GET /v3/api-docs` - Especificação OpenAPI JSON
  - Especificação OpenAPI 3.0 em formato JSON
  - Público: Sim
