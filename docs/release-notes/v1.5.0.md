# Release Notes - Versão 1.5.0

Consolidação de funcionalidades críticas: gerenciamento de badges, announcements (avisos), melhorias em gestão de aulas com upload de arquivos, e novas features de dashboard e ranking para estudantes.

## Visão Geral

Esta versão traz um conjunto significativo de novas funcionalidades educacionais e melhorias administrativas:

- **Badges** (Insígnias): Sistema completo de atribuição e gerenciamento de badges para estudantes
- **Announcements** (Avisos): Endpoints para instrutores criarem e estudantes visualizarem avisos nas turmas
- **Gerenciamento de Aulas**: Melhorias em upload de arquivos e tratamento de tipos de conteúdo
- **Features de Estudante**: Dashboard, seleção de turma melhorada, e ranking de estudantes
- **Correções de Segurança**: Ajustes no contexto de segurança

---

## Funcionalidades Principais

### Badges (Insígnias de Estudante)

**Novos Endpoints:**

- `GET /v1/badges` - Lista badges com paginação (STUDENT)
- `POST /v1/badges/award-retroactive` - Atribui badge retroativamente a estudantes (INSTRUCTOR/ADMIN)

**Novas Entidades e Enums:**

- Entidade `StudentBadge`: vinculação entre estudante e badge com timestamp de aquisição
- Enums `BadgeType` e `BadgeCategory` para categorização de badges

**Detalhes:**

- Sistema de reconhecimento para motivar estudantes
- Suporte a diferentes categorias e tipos de badge
- Endpoint de atribuição retroativa para instrutores compensarem progresso anterior

---

### Announcements (Avisos de Turma)

**Novos Endpoints:**

- `GET /v1/announcements` - Lista avisos de turmas ativas (STUDENT) ou todas (INSTRUCTOR/ADMIN)
- `GET /v1/announcements/{id}` - Obtém detalhes de um aviso específico
- `POST /v1/announcements` - Cria um novo aviso (INSTRUCTOR/ADMIN)
- `PUT /v1/announcements/{id}` - Atualiza um aviso (INSTRUCTOR/ADMIN)
- `DELETE /v1/announcements/{id}` - Exclui um aviso (INSTRUCTOR/ADMIN)

**Documentação:**

- Todos os endpoints com anotações Swagger completas (`@Operation`, `@ApiResponses`, exemplos)
- Suporte a paginação e busca em listagens

---

### Gerenciamento de Aulas (Melhorias)

**Aprimoramentos:**

- Upload de arquivos para aulas com suporte a múltiplos tipos de conteúdo
- Novo utilitário para mapeamento de tipos MIME e categorização de materiais
- Integração com `LessonService` para manipulação de materiais

---

### Features de Estudante

**Novos Endpoints:**

- `POST /v1/auth/select-class` - Estudante seleciona turma
- `GET /v1/student/dashboard` - Dashboard com progresso do estudante
- `GET /v1/student/ranking` - Ranking de estudantes (por pontos, badges, etc.)

**Novas DTOs e Services:**

- `StudentService` com métodos para dashboard, ranking e gestão de progresso
- DTOs para resposta de dashboard e ranking

---

## Detalhes Técnicos

### Entidades Adicionadas

- `StudentBadge`: relacionamento N:M entre `User` (estudante) e badges
- Suporte a timestamp de aquisição para rastreamento histórico

### Enumerações

- `BadgeType`: tipos de badge (ex.: ACHIEVEMENT, STREAK, PARTICIPATION)
- `BadgeCategory`: categorias (ex.: ACADEMIC, BEHAVIORAL)

### Services Adicionados/Modificados

**BadgeService:**

- `awardBadge(studentId, badgeId)` - Atribui badge a estudante
- `getBadgesByStudent(studentId)` - Lista badges de um estudante
- `awardRetroactive(badgeId, studentIds)` - Atribui badge a múltiplos estudantes

**StudentService:**

- `getDashboard(studentId)` - Retorna progresso, pontos, badges recentes
- `getRanking(classroomId)` - Retorna ranking de estudantes da turma
- `selectClass(studentId, classroomId)` - Registra seleção de turma

**ClassroomService:**

- Métodos auxiliares para integração com badges e announcements

### DTOs Adicionados

- `BadgeDTO`: representação de badge
- `StudentBadgeDTO`: vinculação de estudante com badge
- `DashboardDTO`: consolidação de dados do dashboard
- `RankingEntryDTO`: entrada de ranking com posição e pontos
- `AnnouncementDTO`: representação de anúncio

### Repositories Adicionados

- `StudentBadgeRepository`: queries para buscar badges de estudante
- `AnnouncementRepository`: queries para avisos (paginação, filtro por turma, busca)

### Migrations

- **V8__student_badges.sql**: criação de tabelas de badges e relacionamentos

### Documentação Swagger

- Endpoints de announcements e badges com `@Operation`, `@ApiResponses` e exemplos JSON
- Schemas de resposta padronizados (`SuccessResponse`, `ErrorResponse`, `PageResponseDTO`)
