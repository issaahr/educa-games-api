# Release Notes - Versão 1.4.0

Padronização completa da API de módulos, aulas e quizzes com separação de operações, documentação Swagger completa, refatoração de services e migration do banco de dados.

## Visão Geral

Esta versão padroniza completamente a API relacionada a módulos, aulas e quizzes, separando operações de criação e atualização, adiciona documentação Swagger completa com exemplos de respostas, refatora services para melhor separação de responsabilidades, adiciona Javadoc em todos os métodos e cria migration para as novas tabelas do banco de dados.

---

## Funcionalidades Principais

### API de Módulos

**Endpoints padronizados:**

- `GET /v1/module`: Lista módulos com paginação, busca e filtro por curso
- `GET /v1/module/{id}`: Obtém detalhes completos de um módulo
- `POST /v1/module`: Cria novo módulo
- `PUT /v1/module/{id}`: Atualiza título e vínculo com curso
- `DELETE /v1/module/{id}`: Remove módulo

**Endpoints de Aulas:**

- `POST /v1/module/{id}/lessons`: Adiciona novas aulas ao módulo (multipart)
- `PUT /v1/module/{id}/lessons`: Atualiza/substitui aulas do módulo (multipart)

**Endpoints de Quiz:**

- `POST /v1/module/{id}/quiz`: Cria novo quiz para o módulo
- `PUT /v1/module/{id}/quiz`: Atualiza/substitui quiz do módulo

### Padronização de DTOs

Todos os DTOs agora incluem:

- Anotações `@Schema` para documentação OpenAPI
- Validações com mensagens customizadas (`@NotBlank`, `@Size`, `@Min`, `@Max`, `@NotEmpty`)
- DTOs específicos para requests: `AddLessonsRequestDTO`, `UpdateLessonsRequestDTO`

## Detalhes Técnicos

### Refatoração de Services

**ModuleService:**

- Removidas dependências diretas de repositórios de `Lesson` e `Quiz`
- Delegação de responsabilidades para `LessonService` e `QuizService`
- Métodos privados extraídos: `findModuleByIdAndInstructor`, `getLinkedCourseId`, `mapModuleToResponse`, `validateFiles`, `applyFilesToMaterials`
- Javadoc completo em todos os métodos públicos e privados

**LessonService:**

- Métodos públicos expostos: `getLessonsByModule`, `mapLessonsToResponse`, `addLessonsToModule`, `getLessonsByModuleWithMinOrderIndex`, `getMaterialsByLesson`
- Métodos utilitários públicos: `extractYoutubeUrl`, `filterNonYoutubeResources`, `isAllowedLessonContentType`, `mapMaterialType`, `mapContentTypeToMaterialType`, `materialTypeToString`
- Javadoc completo em todos os métodos

**QuizService:**

- Métodos públicos: `hasQuiz`, `getQuizByModule`, `saveQuizFromDto`, `replaceQuizFromDto`
- Javadoc completo em todos os métodos
- Uso de `EntityManager.flush()` e `clear()` para garantir deleção antes de inserção

### Documentação Swagger

Todos os endpoints do `ModuleController` agora incluem:

- `@ApiResponses` com códigos 200, 201, 204, 400, 401, 403, 404
- Schemas de resposta (`SuccessResponse`, `ErrorResponse`)
- Exemplos de resposta para cada código HTTP
- Descrições claras de cada resposta

### Entidades

**Correções:**

- Removidos métodos manuais `getAlternatives()` e `setAlternatives()` de `QuizQuestion` que sobrescreviam Lombok
- Adicionado `@Builder` em `QuizQuestion`
- Adicionado `@Builder.Default` na lista `alternatives`

### Banco de Dados

**Migration V5:**

- Criação de tabelas: `modules`, `lessons`, `lesson_materials`, `quizzes`, `quiz_questions`, `quiz_alternatives`
- Tabela de junção: `course_modules` (many-to-many entre cursos e módulos)
- Tabelas de progresso: `student_module_progress`, `student_lesson_progress`
- Constraints UNIQUE onde necessário
- Foreign keys com `ON DELETE CASCADE`
- Índices para performance
- CHECK constraint para `MaterialType` (LINK, PDF, ZIP, IMAGE)
